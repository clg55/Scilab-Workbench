Scilab keyword

NOM

    overloading - surcharge d'opérateurs, de fonctions ou de l'affichage.

DESCRIPTION

    Dans Scilab, l'affichage des variables, ainsi que les fonctions et les opérateurs peuvent être définis pour de nouveaux objets en utilisant des fonctions (programmées en Scilab ou alors des primitives).

  • Affichage : l'affichage de nouveaux objets définis par les structures de type peut être surchargé (l'affichage par défaut est celui des listes standard). La fonction de surcharge ne doit pas avoir d'argument de sortie et accepter un seul argument d'entrée. Son nom est formé comme ceci : où est la chaîne de caractères contenant le nom du type de la structure (voir ).
  • Opérateurs : chaque opérateur non défini pour des types d'opérandes donnés peut être défini. La fonction de surcharge doit avoir un argument de sortie et un ou deux arguments d'entrée suivant le nombre d'opérandes de l'opérateur choisi. Son nom est formé comme ceci :

    • pour les opérateurs binaires : %<type_opérande_gauche>_<code_opérateur>_<type_opérande_droit>

    pour les opérateurs unaires : %<type_opérande>_<code_opérateur>

    pour les opérateurs d'extraction et d'insertion (opérateurs n-aires)

    <code_opérateur> , <type_opérande_gauche> , <type_opérande_droit> sont des chaînes de caractères associées à chaque type de données comme décrit dans le tableau suivant :

    chaîne c
    polynôme p
    fonction m
    constante s
    list l
    tlist <tlist_type>
    booléens b
    matrice creuse sp
    matrice creuse booléenne spb

    <code_opérateur> est un caractère associé à chaque type d'opérateur comme décrit dans ce tableau :

    ' t
    + a
    - s
    * m
    / r
    \ l
    ^ p
    .* x
    ./ d
    .\ q
    .*. k
    ./. y
    .\. z
    : b
    *. u
    /. v
    \. w
    [a,b] c
    [a;b] f
    () extraction e
    () insertion i
    == o
    <> n
    | g
    & h
    .^ j
    ~ 5
    .' 0
    < 1
    > 2
    <= 3
    >= 4

    La fonction de surcharge pour la syntaxe d'extraction b=a(i1,...,in) a la séquence d'appel suivante : b=%<type_de_a>_e_(i1,...,in,a)

    et la syntaxe [x1,..,xm]=a(i1,...,in) a la séquence d'appel suivante : [x1,..,xm]=%<type_de_a>_e_(i1,...,in,a)

    La fonction de surcharge pour la syntaxe d'insertion a(i1,...,in)=b a la séquence d'appel suivante : a=%<type_de_a>_i_<type_de_b>(i1,...,in,b,a) .

  • Fonctions : certaines primitives peuvent aussi être surchargées pour de nouveaux types d'objets. Quand une telle fonction n'est pas définie pour un type d'objet particulier la fonction est appelée. L'utilisateur peut ajouter dans cette fonction la définition associée avec le type de données passées en argument d'entrée.

EXEMPLES

    // AFFICHAGE
deff('[]=%tab_p(l)','disp([['' '';l(3)] [l(2);string(l(4))]])')
tlist('tab',['a','b'],['x';'y'],rand(2,2))

// OPÉRATEUR
deff('x=%c_a_s(a,b)','x=a+string(b)')
's'+1

// FONCTION
deff('x=%c_sin(a)','x=''sin(''+a+'')''')
sin('2*x')
